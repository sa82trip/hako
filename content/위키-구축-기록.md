이 문서는 **이 위키(箱・はこ)** 가 어떻게 만들어지고 배포되는지를 기록한 참고용 노트다. 나중에 세팅을 다시 보거나 고칠 때 참고하려고 썼다.

## 목표

- 검색이 되고 **백링크**가 되는 개인 위키
- **아주 단순**할 것 — "어떻게 돌아가는지 모르면 결국 안 쓰게 된다"
- 비용이 들지 않을 것

## 왜 이 스택인가

| 구성                 | 역할                                 |
| ------------------ | ---------------------------------- |
| **Obsidian**       | 로컬 글쓰기 에디터. 마크다운 + 백링크             |
| **Quartz v4**      | 마크다운을 정적 웹사이트로 변환 (TypeScript SSG) |
| **GitHub Pages**   | 무료 호스팅 (public 저장소)                |
| **GitHub Actions** | push할 때마다 자동 빌드 + 배포               |

Quartz는 [[백링크-테스트|백링크]]와 전문 검색을 **브라우저 안에서** 처리한다. 백엔드 서버가 필요 없어서 무료 호스팅과 찰떡이다. 기본 문법은 [[Markdown-문법]]을, 전체 안내는 [[시작하기]]를 참고.

## 세팅 과정

### 1. Quartz 설치
- 처음엔 Quartz **v5**를 시도했으나 아직 unstable (`.scss` 확장자 에러 발생) → **v4.5.2**로 안정화.
- Node 22 필요 → `.nvmrc`에 `"22"` 고정.

### 2. 설정 (`quartz.config.ts`)
- `pageTitle`: "箱(はこ)"
- `locale`: `ko-KR`
- `baseUrl`: `sa82trip.github.io/hako`
- analytics 블록 제거, `CustomOgImages`는 Google Fonts fetch 실패로 주석 처리.

### 3. GitHub 저장소 + Pages
- 저장소 `sa82trip/hako` (**public** — Pages 무료 조건).
- fine-grained token엔 저장소 생성 권한이 없어서, 저장소는 GitHub 웹에서 직접 생성.
- `.github/workflows/deploy.yml` 추가: `main` push → `npm ci` + `npx quartz build` → Pages 배포.

### 4. 첫 배포 (트러블슈팅)
- **첫 빌드**: ✅ build 성공 / ❌ deploy 실패.
- **원인**: GitHub Pages가 아직 활성화되지 않음 (HTTP 404).
- **해결**: Settings → Pages → Source를 **"GitHub Actions"** 로 변경.
- 빈 커밋 push로 재배포 트리거 → ✅ build + deploy 모두 성공.
- **결과**: https://sa82trip.github.io/hako 라이브.

## 자동화

### 글 쓰면 자동 배포 (Obsidian Git)
- 커뮤니티 플러그인 **Obsidian Git** 설치.
- **5분마다** 변경사항이 있으면 자동 commit + push.
- push 인증은 `gh` credential helper가 처리 → 비밀번호 입력 불필요.
- 흐름: `글 쓰기 → (5분마다) 자동 push → Actions 자동 빌드/배포 → 사이트 반영`.

### 미완성 글 보호 (draft)
- Quartz에 `RemoveDrafts` 필터가 켜져 있어, frontmatter에 `draft: true`가 있으면 **사이트에 노출되지 않는다**.
- 작업 중인 글은 `draft: true`로 시작하고, 다 쓰면 그 줄을 지워 공개.
- `content/templates/note.md` 템플릿에 `draft: true`가 미리 들어있다.

## 비용

전부 **무료**. 유일한 조건은 **저장소를 public으로 유지**하는 것뿐.
- GitHub 저장소·Pages·Actions(public) = 무료·무제한
- Quartz·Obsidian(개인용) = 무료
- 현실적 제한: 저장소 1GB, 월 대역폭 100GB (개인 위키로는 평생 닿지 않음)

> private으로 바꾸면 Pages가 유료(GitHub Pro)가 되므로 주의.

## 이미지 관리

- 스크린샷 캡처(`Ctrl+Cmd+Shift+4`) → Obsidian에 붙여넣기 → **Image Converter** 플러그인이 자동으로 WebP 변환 + 압축 + 리사이즈.
- 폴더 프리셋을 `images/{noteName}`으로 설정해 **노트별 하위폴더**로 자동 정리.
- 오랜 기간 쓰면 git 히스토리에 과거 이미지가 누적되므로, 가끔 **히스토리 압축(squash)** 으로 회수.

## 요약

```
Obsidian으로 글을 쓴다
   → Obsidian Git이 자동 commit + push
   → GitHub Actions가 자동 빌드 + 배포
   → 사이트에 반영
```

핵심은 **"글만 쓰면 끝"** 인 구조. 나머지는 전부 자동이다.
